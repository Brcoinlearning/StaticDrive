## 需求概述

### 项目名称
本地文件管理与发布服务

### 项目描述
运行于1panel管理的MiniPC上的文件管理服务，为本地智能体"书童四九"提供文件存储能力，并通过静态网页服务向外部用户提供HTTP访问。

![image-20260414151837362](/Users/mr.hu/Library/Application Support/typora-user-images/image-20260414151837362.png)

### 项目背景
- **部署环境**：本地MiniPC，由1panel管理
- **使用场景**：本地智能体"书童四九"调用云端大模型API并生成文件，需要存储并提供HTTP访问
- **系统定位**：大项目"书童四九"的子功能模块，需要预留用户系统接口

### 系统架构

```
书童四九智能体
    ↓ (HTTP API)
文件管理服务
    ↓ (工作目录)
静态网页服务
    ↓ (HTTP)
外部用户
```

### 项目目标
- 为智能体提供文件存储API
- 为外部用户提供HTTP访问能力
- 防止越权访问（URL hash处理）
- 支持内容搜索与管理

## 功能需求

### FR-001: 文件上传与存储
**优先级**: Must

**描述**: 接收书童四九智能体通过API上传的文件，存储到工作目录

**验收标准**:
- 接收HTTP API上传的文件
- 文件按原文件名存储在本地工作目录
- 为每个文件生成唯一hash标识
- 保存文件元数据（文件名、大小、上传时间、所属用户等）

---

### FR-002: 富文本保存与查看
**优先级**: Must

**描述**: 接收书童四九智能体保存的富文本内容，支持HTML渲染展示

**验收标准**:
- 接收富文本内容（HTML格式）
- 保存富文本及其元数据（标题等）
- 支持以HTML形式渲染展示
- 富文本内容可被搜索

---

### FR-003: 内容列表展示
**优先级**: Must

**描述**: 展示用户发布的内容列表，支持分页和排序

**验收标准**:
- 用户只能查看自己发布的内容列表
- 支持分页展示
- 支持按创建时间排序
- 支持按文件名排序
- 列表项显示文件名、上传时间、文件大小

---

### FR-004: 内容搜索
**优先级**: Must

**描述**: 支持按文件名/标题搜索发布的内容

**验收标准**:
- 支持按文件名搜索（必须）
- 支持按富文本标题搜索（可选）
- 提交式搜索（非实时）
- 搜索结果高亮匹配关键字

---

### FR-005: 内容详情查看
**优先级**: Must

**描述**: 查看单个内容的详细信息，并提供下载访问

**验收标准**:
- 展示文件名、文件大小
- 富文本内容支持HTML渲染展示
- 其他文件支持下载
- URL使用hash标识（如 /files/a7f8c9b2d3e4f5.html）

---

### FR-006: 内容管理（CRUD）
**优先级**: Must

**描述**: 用户对自己发布的内容有完整的增删改查权限

**验收标准**:
- 创建：上传文件/保存富文本
- 读取：查看列表、搜索、详情
- 更新：修改元数据/内容（待明确）
- 删除：删除自己的内容

---

### FR-007: 分享链接生成
**优先级**: Must

**描述**: 为内容生成可分享的URL，允许外部用户访问

**验收标准**:
- 生成包含hash的分享URL
- 外部用户通过分享URL可访问内容
- 支持撤销分享链接

---

### FR-008: Open API接口
**优先级**: Must

**描述**: 提供HTTP API供书童四九智能体调用

**验收标准**:
- 保存文件API
- 保存富文本API
- 获取内容列表API
- 搜索内容API
- 删除内容API
- 使用API Key进行身份识别

---

### FR-009: 访问次数统计
**优先级**: Could（非MVP）

**描述**: 统计内容的访问次数

**验收标准**:
- 记录每次访问
- 在列表和详情页展示访问次数

---

### FR-010: 文件生命周期管理
**优先级**: Should

**描述**: 支持文件自动过期、手动删除、空间限制管理

**验收标准**:
- 支持自动过期删除

## 业务规则

### BR-001: 内容归属规则
所有内容必须归属于某个用户（通过API Key识别）

**条件**: 上传文件时必须提供有效的API Key
**动作**:
- 将内容关联到对应用户
- 用户只能查看和管理自己的内容

---

### BR-002: 访问权限规则
用户对自己内容有完整权限，对别人内容只能通过分享链接访问

**条件**: 访问自己的内容
**动作**: 允许增删改查操作

**条件**: 访问别人的内容
**动作**:
- 只能通过有效的分享链接访问
- 只能查看和下载，不能修改

---

### BR-003: URL Hash规则
对外URL使用hash标识，不暴露原始文件名

**条件**: 生成访问URL
**动作**:
- 为内容生成唯一hash
- URL格式：/files/{hash}.{ext} 或 /share/{hash}
- 本地存储仍使用原文件名

---

### BR-004: 分享链接规则
分享链接支持多种安全配置

**条件**: 生成分享链接
**动作**:
- 发布者可选择一次性或永久有效
- 支持设置过期时间（MVP以外）
- 支持设置访问密码（MVP以外）
- 支持撤销分享链接

---

### BR-005: 搜索规则
搜索基于文件名和标题

**条件**: 用户执行搜索
**动作**:
- 必须匹配文件名
- 可选匹配富文本标题
- 优先标题匹配
- 高亮显示匹配关键字

## 数据规范

### User（用户）
| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| user_id | string | 唯一标识 | 用户ID |
| api_key | string | 唯一 | 用于API调用认证 |

**验证规则**:
- API Key必须唯一
- API Key格式待定

---

### Content（内容）
| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| content_id | string | 唯一标识 | 内容ID |
| user_id | string | 外键 | 关联到用户 |
| type | enum | - | file | rich_text |
| title | string | - | 标题/文件名 |
| original_filename | string | - | 原始文件名（file类型） |
| file_path | string | - | 本地存储路径 |
| content_hash | string | 唯一 | 用于URL的hash标识 |
| file_size | integer | 非负 | 文件大小（字节） |
| content | text | - | 富文本内容（rich_text类型） |
| metadata | json | - | 扩展元数据 |
| access_count | integer | 非负 | 访问次数（非MVP） |
| created_at | datetime | - | 创建时间 |
| updated_at | datetime | - | 更新时间 |

**验证规则**:
- content_id必须唯一
- user_id必须有效
- file_size必须非负

---

### ShareLink（分享链接）
| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| share_id | string | 唯一标识 | 分享ID |
| content_id | string | 外键 | 关联到内容 |
| share_hash | string | 唯一 | 分享链接hash |
| is_one_time | boolean | - | 是否一次性 |
| expire_at | datetime | - | 过期时间（MVP以外） |
| password | string | - | 访问密码（MVP以外） |
| is_revoked | boolean | - | 是否已撤销 |
| created_at | datetime | - | 创建时间 |

**验证规则**:
- 过期后不能访问
- 撤销后不能访问

## 边界条件

| 场景 | 处理方式 |
|------|----------|
| 访问不存在的hash URL | 返回404错误 |
| 访问已撤销的分享链接 | 返回分享已失效提示 |
| 访问已过期的分享链接 | 返回分享已过期提示（MVP以外） |
| 使用无效的API Key调用API | 返回401认证失败 |
| 用户访问别人的内容（无分享链接） | 返回403无权限 |
| 文件名包含特殊字符 | URL编码处理 |
| 上传超大文件 | 待明确：大小限制和处理方式 |

---

## 异常处理

| 场景 | 处理方式 |
|------|----------|
| API调用失败 | 返回明确的错误信息 |
| 文件存储失败（磁盘空间不足等） | 返回错误信息，不保存内容 |
| 数据库操作失败 | 返回错误信息，回滚操作 |
| hash生成冲突（极低概率） | 重新生成hash |

## MVP范围

### 包含功能
- FR-001: 文件上传与存储
- FR-002: 富文本保存与查看
- FR-003: 内容列表展示（不含访问次数排序）
- FR-004: 内容搜索（文件名）
- FR-005: 内容详情查看
- FR-006: 内容管理（CRUD）
- FR-007: 分享链接生成（基础版，不含密码和过期）
- FR-008: Open API接口
- URL hash处理
- API Key认证
- 基础权限控制

### 不包含功能
- 访问次数统计（MVP以后）
- 分享链接密码保护（MVP以后）
- 分享链接过期时间（MVP以后）
- 高级生命周期管理（MVP以后）
- 用户系统（预留接口）
