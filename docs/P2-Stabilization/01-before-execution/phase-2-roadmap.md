# Phase 2 Roadmap

本文档汇总 MVP 完成后的下一阶段工作方向，属于“下个阶段”规划文档。

## 1. 目标定位

当前 MVP 已完成“写入 -> 查询 -> 分享访问 -> 网页展示 -> 撤销/删除”的主链路。

Phase 2 的目标不是重复建设 MVP，而是把现有能力从“可演示、可验证”推进到“更稳定、可持续扩展、适合长期使用”。

## 2. 优先级

### P0 稳定化

- 增加启动前自检
- 改善 PocketBase 失败时的错误提示
- 做一次从空环境开始的冷启动验证
- 固化部署步骤与运维注意事项

### P1 可运维性与体验

- 增加更细的运行状态检查
- 改进日志可读性
- 优化 owner 访问体验
- 优化文件下载体验

### P2 安全与产品增强

- API Key 管理能力
- 审计记录
- 输入校验增强
- 内容更新能力
- 批量操作与搜索增强

## 3. 建议执行顺序

1. 先补稳定化与冷启动验证
2. 再补 owner 访问体验与文件下载体验
3. 然后进入安全与产品能力扩展
4. 最后补更多真实环境集成测试
