# 业务规则备忘录

## 1. 需求概述

### 项目名称

本地文件管理与发布服务

### 项目描述

运行于 1Panel 管理的 MiniPC 上，为本地智能体“书童四九”提供文件存储与内容发布能力，并通过静态网页服务向外部用户提供 HTTP 访问。

### 项目背景

- 部署环境为本地 MiniPC，由 1Panel 管理。
- 本地智能体“书童四九”调用云端大模型 API 并在本地生成文件或 HTML 内容。
- 系统是“书童四九”大项目中的子模块，需要为后续用户系统预留接口。

## 2. 核心目标

- 为智能体提供可调用的文件与富文本写入 API。
- 为外部访问者提供只读的 HTTP 查看或下载能力。
- 在对外访问链路中引入 hash 标识，避免直接暴露原始文件名。
- 提供内容列表、搜索、详情和基础管理能力。

## 3. 范围内内容

- 文件上传与落盘。
- HTML 富文本保存与渲染查看。
- 内容元数据保存。
- 内容列表、分页、排序、搜索、详情查看。
- 基础 CRUD 能力。
- 分享链接生成与撤销。
- Open API 接入，使用 API Key 识别调用方。
- 用户内容归属与基础权限控制。

## 4. 范围外内容

- 外网访问策略与公网暴露方案。
- 完整用户系统实现，仅预留接口。
- 访问密码保护、分享链接过期时间等增强安全配置。
- 访问次数统计。
- 高级生命周期管理和大文件专项优化。

## 5. 核心概念定义

- `书童四九`：运行于本地 MiniPC 的智能体，负责生成文件或 HTML 内容并调用本系统 API。
- `文件管理服务`：本系统本体，负责接收内容、存储元数据、管理权限并提供 API。
- `静态网页服务`：对外提供内容查看或下载入口的只读 HTTP 层。
- `工作目录`：文件在本地物理机上的实际存储目录。
- `分享链接`：外部用户访问具体内容的 URL，使用 hash 标识而非原始文件名。
- `API Key`：用于标识调用用户身份的密钥。

## 6. 用户与使用场景

### 用户角色

- 发布者：通过 API Key 调用系统的智能体或其代表用户。
- 外部访问者：通过分享链接访问内容的只读用户。

### 典型场景

- 智能体生成文件后通过 Open API 上传，系统落盘并生成可访问资源。
- 智能体生成 HTML 富文本后通过 Open API 保存，系统提供渲染查看页。
- 发布者在自己的内容范围内查看列表、搜索内容、查看详情、删除内容。
- 外部用户通过分享链接访问单个内容。

## 7. 功能性要求

### FR-001 文件上传与存储

- 接收 HTTP API 上传的文件。
- 文件按原文件名存储到本地工作目录。
- 为每个文件生成唯一 hash 标识。
- 保存文件元数据，包括文件名、大小、上传时间、所属用户等。

### FR-002 富文本保存与查看

- 接收 HTML 富文本内容。
- 保存富文本内容及其标题等元数据。
- 支持 HTML 渲染展示。
- 富文本内容可被搜索。

### FR-003 内容列表展示

- 仅展示当前用户发布的内容列表。
- 支持分页。
- 支持按创建时间排序。
- 支持按文件名排序。
- 列表项显示文件名、上传时间、文件大小。

### FR-004 内容搜索

- 支持按文件名搜索。
- 可选支持按富文本标题搜索。
- 使用提交式搜索，而非实时搜索。
- 搜索结果支持高亮匹配关键字。

### FR-005 内容详情查看

- 展示文件名、文件大小等基本信息。
- 富文本内容支持 HTML 渲染。
- 其他文件支持下载。
- 对外 URL 使用 hash 标识，例如 `/files/a7f8c9b2d3e4f5.html`。

### FR-006 内容管理

- 创建：上传文件或保存富文本。
- 读取：查看列表、搜索、详情。
- 更新：允许修改元数据，内容编辑能力暂未细化。
- 删除：删除自己的内容。

### FR-007 分享链接生成

- 为内容生成包含 hash 的访问 URL。
- 外部用户可通过该 URL 访问内容。
- 支持撤销分享链接。

### FR-008 Open API 接口

- 提供保存文件 API。
- 提供保存富文本 API。
- 提供获取内容列表 API。
- 提供搜索内容 API。
- 提供删除内容 API。
- 使用 API Key 进行身份识别。

## 8. 非功能性要求

### 安全性

- 使用 API Key 识别调用方身份。
- 对外 URL 使用 hash 标识，避免暴露原始文件名。
- 用户只能查看和管理自己的内容。
- 别人的内容只能通过有效分享链接访问。

### 可扩展性

- 预留未来用户系统对接能力。
- 支持后续补充访问统计、生命周期管理和大文件能力。

### 兼容性

- 适配 1Panel 管理下的 MiniPC 单机部署场景。
- 与“书童四九”主项目通过 HTTP API 集成。

### 可靠性

- 文件落盘与元数据写入需要保持一致性。
- 数据库或存储失败时应返回明确错误并中止本次写入。

## 9. 业务规则

### BR-001 内容归属规则

所有内容必须归属于某个用户，用户身份通过 API Key 识别。

### BR-002 访问权限规则

用户对自己内容拥有完整权限；访问他人内容时，仅允许通过有效分享链接进行只读访问。

### BR-003 URL Hash 规则

对外访问 URL 使用 hash 标识，不暴露原始文件名；本地存储仍可保留原始文件名。

### BR-004 分享链接规则

MVP 需支持基础分享链接生成与撤销；一次性访问、过期时间、访问密码属于后续增强能力。

### BR-005 搜索规则

搜索以文件名为必选匹配项，可扩展支持富文本标题匹配，并优先展示标题命中结果。

## 10. 边界条件

- 访问不存在的 hash URL 时返回 `404`。
- 访问已撤销的分享链接时返回“分享已失效”。
- 使用无效 API Key 调用 API 时返回 `401`。
- 用户访问他人内容但未携带有效分享链接时返回 `403`。
- 文件名包含特殊字符时，需要进行 URL 编码或内部映射处理。
- 大文件上传存在潜在大小限制，但当前未定义具体阈值。

## 11. 异常处理

- API 调用失败时返回明确错误信息。
- 文件存储失败时不应保留不完整元数据。
- 数据库操作失败时应回滚当前事务性写入。
- hash 冲突时重新生成 hash。

## 12. 数据约束

### User

- `user_id` 必须唯一。
- `api_key` 必须唯一。

### Content

- `content_id` 必须唯一。
- `user_id` 必须关联有效用户。
- `type` 取值为 `file` 或 `rich_text`。
- `content_hash` 必须唯一。
- `file_size` 必须为非负整数。

### ShareLink

- `share_id` 必须唯一。
- `content_id` 必须关联有效内容。
- `share_hash` 必须唯一。
- 已撤销的分享链接不能继续访问。

## 13. 待确认事项

- API Key 的生成、轮换、过期和多 Key 策略。
- 生命周期管理策略，包括自动过期和空间限制。
- 富文本编辑更新是否纳入 MVP。
- 列表分页大小和默认排序规则。
- 大文件处理策略，包括上限、流式传输和断点续传。
- 性能指标，如目标响应时间和并发量。
