# P3-T2 Owner Frontend 人工验收

本文档用于人工验收 Phase 3 Task 2 的 owner 前端产品化页面。

本轮人工验收只覆盖 P3-T2 冻结后的范围：

1. owner 列表页
2. owner 搜索页
3. owner 详情页
4. 单条分享
5. 单条撤销分享
6. 单条删除
7. 页面级成功 / 失败反馈

不覆盖：

- 批量操作
- owner 登录体系
- 富文本在线编辑
- API Key 生命周期管理

owner 页面仍然通过 API Key 访问，因此建议使用浏览器插件、调试工具或 curl 先验证 header。

浏览器访问 owner 页面时，请确保请求头带上：

```text
x-shutong49-api-key: t1_verify_api_key_0001
```

- `http://127.0.0.1:8787/web/list`
- `http://127.0.0.1:8787/web/search?q=试卷`

## 4. 验收步骤

### 4.1 列表页

打开：

```text
http://127.0.0.1:8787/web/list
```

确认点：

- 页面标题为 `Owner 内容列表`
- 能看到 owner 已写入内容
- 每条内容卡片至少展示：标题、类型、共享状态、MIME、大小
- 卡片上有 `查看详情`
- 已分享内容可看到 `打开公开页`
- 未分享内容显示不可公开访问提示

### 4.2 搜索页

打开：

```text
http://127.0.0.1:8787/web/search?q=试卷
```

确认点：

- 页面标题为 `Owner 搜索结果`
- 查询词会回填到搜索框
- 结果只显示 owner 自己的内容
- 没有命中时显示明确 empty 状态

### 4.3 详情页

从列表页点击任一内容进入，或直接打开：

```text
http://127.0.0.1:8787/web/detail/<contentId>
```

确认点：

- 页面显示该内容的摘要信息
- 富文本内容可在 iframe 中预览
- 文件内容显示文件名、MIME、大小
- 页面右侧有 `Owner 操作` 区
- 至少有分享/撤销分享和删除入口之一

### 4.4 单条分享

在未公开内容详情页点击 `创建分享`。

确认点：

- 提交后页面发生重定向
- 页面顶部出现成功反馈：`分享已创建`
- 内容状态从未公开变为已公开
- 页面出现公开访问入口
- 公开页可以正常打开

### 4.5 单条撤销分享

在已公开内容详情页点击 `撤销分享`。

确认点：

- 提交后页面发生重定向
- 页面顶部出现成功反馈：`分享已撤销`
- 内容状态从已公开变为未公开
- 原公开页应返回不可访问或失效状态

### 4.6 单条删除

在详情页点击 `删除内容`。

确认点：

- 浏览器出现确认提示
- 确认后跳回列表页
- 页面顶部出现成功反馈：`内容已删除`
- 列表中不再出现该内容
- 若该内容为文件，其公开下载页不再可用

### 4.7 错误反馈

可以用以下方式验证失败反馈：

- 对已删除内容重复访问详情页
- 对不存在的 contentId 手工访问详情页
- 在未带 API Key 的情况下访问 owner 页面

确认点：

- 页面能显示明确错误页，而不是空白页
- 错误信息可读
- 未带 API Key 时仍会先走鉴权失败

## 5. 推荐演示顺序

建议现场顺序：

1. 打开 `/web/list`
2. 从列表进入某个内容详情页
3. 在详情页执行分享或撤销分享
4. 打开对应公开页验证结果
5. 返回详情页执行删除
6. 回到列表确认内容消失
7. 再执行一次搜索页演示

## 6. 通过标准

满足以下条件可判定 P3-T2 人工验收通过：

- owner 列表、搜索、详情三个页面都可正常访问
- 单条分享、撤销分享、删除都能通过浏览器页面完成
- 页面成功 / empty / error 状态都有明确反馈
- 未出现越界到批量操作或登录体系的新入口
