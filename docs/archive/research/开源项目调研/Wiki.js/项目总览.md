## 项目简介

Wiki.js 是一个现代化、强大且可扩展的 Wiki 软件，采用 Node.js 构建。它提供了一个直观的界面来创建、组织和管理知识库内容，同时支持多种编辑器和存储后端。

## 核心特性

### 1. 多编辑器支持
- **Markdown**: 标准 Markdown 编辑器，支持 GFM
- **Code (MDX)**: 支持 JavaScript 组件的增强 Markdown
- **Visual Editor**: 所见即所得的富文本编辑器
- **AsciiDoc**: 技术文档标准格式
- **PlantUML**: UML 图表编辑
- **Mermaid**: 流程图和图表

### 2. 强大的权限系统
- **基于角色的访问控制 (RBAC)**
- **页面级权限规则**
- **细粒度权限设置**
- **多种认证提供者**（LDAP, OAuth2, SAML等）

### 3. 灵活的存储选项
- **本地文件系统**
- **Amazon S3**
- **Azure Storage**
- **Google Cloud Storage**
- **其他 S3 兼容存储**

### 4. 资产管理
- **拖放上传**
- **文件夹组织**
- **图片优化**
- **访问控制**
- **CDN 集成**

### 5. 多语言支持
- **完整的本地化系统**
- **多语言内容版本**
- **RTL 语言支持**
- **翻译管理**

### 6. 企业级功能
- **版本控制**
- **活动日志和审计**
- **备份和恢复**
- **API 访问**
- **Webhook 集成**

## 技术栈

### 后端
- **Node.js**: 服务器运行时

### 前端
- **Vue.js**: 前端框架
- **Vuetify**: Material Design 组件库
- **Ace Editor**: 代码编辑器
- **Graphviz**: 图表可视化

### 数据库支持
- **PostgreSQL** (推荐)
- **MySQL / MariaDB**
- **SQLite** (开发/小型部署)
- **Microsoft SQL Server**

## 安装方式

### Docker 安装（推荐）
```bash
docker run -d -p 80:3000 \
  -e "DB_TYPE=postgres" \
  -e "DB_HOST=db" \
  -e "DB_PORT=5432" \
  -e "DB_USER=wikijs" \
  -e "DB_PASS=wikijsrocks" \
  -e "DB_NAME=wiki" \
  --name wiki \
  requarks/wiki:2
```

### Node.js 直接安装
```bash
wget https://github.com/Requarks/wiki/releases/latest/download/wiki-js.tar.gz
tar xzf wiki-js.tar.gz
npm install --production
node server
```

## 与其他 Wiki 软件比较

### vs. Confluence
- **优势**: 开源免费、更轻量、更灵活
- **劣势**: 企业级功能相对较少
- **适用**: 中小企业、初创公司

### vs. MediaWiki
- **优势**: 更现代的界面、更好的用户体验
- **劣势**: 插件生态相对较小
- **适用**: 现代化团队、技术团队

### vs. Notion
- **优势**: 自托管、数据控制权、可定制
- **劣势**: 协作功能相对简单
- **适用**: 注重数据隐私的组织

### vs. BookStack
- **优势**: 更强大的编辑器、更好的 API
- **劣势**: 界面相对复杂
- **适用**: 技术团队、开发者

## 部署建议

### 小型团队（< 50 用户）
- **硬件**: 2 核 CPU, 2GB RAM
- **数据库**: SQLite 或 PostgreSQL
- **存储**: 本地文件系统
- **备份**: 每日自动备份

### 中型团队（50-200 用户）
- **硬件**: 4 核 CPU, 4GB RAM
- **数据库**: PostgreSQL
- **存储**: 云存储（S3 等）
- **缓存**: Redis
- **负载均衡**: Nginx 反向代理

### 大型企业（200+ 用户）
- **硬件**: 8+ 核 CPU, 8GB+ RAM
- **数据库**: PostgreSQL 集群
- **存储**: 企业级云存储
- **缓存**: Redis 集群
- **高可用**: 负载均衡 + 容器编排
