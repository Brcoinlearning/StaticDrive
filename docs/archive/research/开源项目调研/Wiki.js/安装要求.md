# Wiki.js 安装要求和配置文档

## 系统要求

### 最低硬件要求

#### CPU
- **最低**: 1 核心处理器
- **推荐**: 2+ 核心处理器
- **高性能**: 4+ 核心处理器（适用于大型部署）

#### 内存 (RAM)
- **最低**: 512MB RAM
- **推荐**: 1GB RAM
- **生产环境**: 2GB+ RAM
- **大型部署**: 4GB+ RAM

#### 存储空间
- **最低安装**: 500MB 可用空间
- **推荐**: 5GB+ 可用空间
- **生产环境**: 20GB+ SSD 存储
- **大型部署**: 50GB+ SSD 存储

### 操作系统支持

Wiki.js 可在以下操作系统上运行：

#### Linux
- **Ubuntu**: 18.04 LTS, 20.04 LTS, 22.04 LTS
- **Debian**: 9, 10, 11
- **CentOS**: 7, 8, 9 Stream
- **RHEL**: 7, 8, 9
- **Fedora**: 最新稳定版
- **Alpine Linux**: 3.12+

#### Windows
- **Windows Server**: 2016, 2019, 2022
- **Windows Desktop**: 10, 11（开发环境）

#### macOS
- **macOS**: 10.15+（开发环境）

### 软件依赖

#### Node.js
```bash
# 必需的 Node.js 版本
Node.js: >= 14.x
推荐: 18.x LTS 或 20.x LTS

# 检查 Node.js 版本
node -v

# 推荐使用版本管理器
nvm install 18
```

#### 包管理器
```bash

npm >= 6.x

# yarn (可选但推荐)
npm install -g yarn

# pnpm (可选)
npm install -g pnpm
```

#### 数据库引擎
Wiki.js 支持以下数据库：

##### PostgreSQL
```bash
# 支持版本
PostgreSQL: >= 9.5
推荐: 12.x 或更高版本

# 安装示例 (Ubuntu)
sudo apt-get install postgresql postgresql-contrib

# 创建数据库
sudo -u postgres createdb wiki
sudo -u postgres psql
CREATE USER wikijs WITH PASSWORD 'yourpassword';
GRANT ALL PRIVILEGES ON DATABASE wiki TO wikijs;
```

##### MySQL / MariaDB
```bash
# 支持版本
MySQL: >= 5.7
MariaDB: >= 10.2

# 安装示例 (Ubuntu)
sudo apt-get install mysql-server

# 创建数据库
mysql -u root -p
CREATE DATABASE wiki;
CREATE USER 'wikijs'@'localhost' IDENTIFIED BY 'yourpassword';
GRANT ALL PRIVILEGES ON wiki.* TO 'wikijs'@'localhost';
FLUSH PRIVILEGES;
```

##### SQLite
```bash
# 适用场景
- 开发环境
- 小型部署
- 测试环境

# SQLite 支持
SQLite: >= 3.x
通常无需单独安装
```

##### Microsoft SQL Server
```bash
# 支持版本
SQL Server: >= 2012
SQL Server Express: >= 2012

# 连接字符串示例
Server=localhost;Database=wiki;User Id=wikijs;Password=yourpassword;
```

## 安装方法

### 方法 1: Docker 安装（推荐）

#### 使用 Docker Compose
```yaml
version: "3"
services:
  wiki:
    image: requarks/wiki:2
    ports:
      - "80:3000"
    environment:
      - DB_TYPE=postgres
      - DB_HOST=db
      - DB_PORT=5432
      - DB_USER=wikijs
      - DB_PASS=wikijsrocks
      - DB_NAME=wiki
    depends_on:
      - db
    restart: unless-stopped

  db:
    image: postgres:15-alpine
    environment:
      - POSTGRES_USER=wikijs
      - POSTGRES_PASSWORD=wikijsrocks
      - POSTGRES_DB=wiki
    volumes:
      - db-data:/var/lib/postgresql/data
    restart: unless-stopped

volumes:
  db-data:
```

#### 单独运行 Docker 容器
```bash
docker run -d -p 80:3000 \
  -e "DB_TYPE=postgres" \
  -e "DB_HOST=your-db-host" \
  -e "DB_PORT=5432" \
  -e "DB_USER=wikijs" \
  -e "DB_PASS=yourpassword" \
  -e "DB_NAME=wiki" \
  --name wiki \
  requarks/wiki:2
```

### 方法 2: Node.js 直接安装

#### 下载和安装
```bash
# 1. 创建项目目录
mkdir wiki
cd wiki

# 2. 下载最新版本
wget https://github.com/Requarks/wiki/releases/latest/download/wiki-js.tar.gz

# 3. 解压文件
tar xzf wiki-js.tar.gz

# 4. 安装依赖
npm install --production

# 5. 配置 Wiki.js
cp config.sample.yml config.yml
nano config.yml

# 6. 启动服务
node server
```

#### 使用 PM2 管理进程
```bash
# 安装 PM2
npm install -g pm2

# 启动 Wiki.js
pm2 start server.js --name wiki

# 设置开机自启
pm2 startup
pm2 save

# 常用命令
pm2 status wiki
pm2 logs wiki
pm2 restart wiki
pm2 stop wiki
```

### 方法 3: 从源代码构建

```bash
# 1. 克隆仓库
git clone https://github.com/Requarks/wiki.git
cd wiki

# 2. 安装开发依赖
npm install

# 3. 构建项目
npm run build

# 4. 启动开发服务器
npm run dev

# 5. 生产构建
npm run build:production
node server
```

## 配置文件

### config.yml 结构
```yaml
# 服务器配置
port: 3000
bindIP: '0.0.0.0'

# 数据库配置
db:
  type: postgres
  host: localhost
  port: 5432
  user: wikijs
  pass: wikijsrocks
  db: wiki
  ssl: false

# 日志配置
logLevel: info

# 数据路径
dataPath: ./data

# 上传限制
uploads:
  maxFileSize: 5242880
  maxFiles: 10

# SSL 配置
ssl:
  enabled: true
  provider: letsencrypt
  domain: wiki.example.com

# 邮件配置
mail:
  enabled: true
  host: smtp.example.com
  port: 465
  secure: true
  user: noreply@example.com
  pass: password
  from: Wiki.js <noreply@example.com>

# OAuth 配置
oauth:
  google:
    enabled: true
    clientId: YOUR_CLIENT_ID
    clientSecret: YOUR_CLIENT_SECRET

# 存储配置
storage:
  type: local
  local:
    path: ./data/uploads

# SEO 配置
seo:
  enabled: true
  robots:
    allow: true
    sitemap: true
```

## 反向代理配置

### Nginx 配置
```nginx
server {
    listen 80;
    server_name wiki.example.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name wiki.example.com;

    ssl_certificate /etc/letsencrypt/live/wiki.example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/wiki.example.com/privkey.pem;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

### Apache 配置
```apache
<VirtualHost *:80>
    ServerName wiki.example.com
    Redirect permanent / https://wiki.example.com/
</VirtualHost>

<VirtualHost *:443>
    ServerName wiki.example.com
    SSLEngine on
    SSLCertificateFile /etc/letsencrypt/live/wiki.example.com/fullchain.pem
    SSLCertificateKeyFile /etc/letsencrypt/live/wiki.example.com/privkey.pem

    ProxyPreserveHost On
    ProxyRequests Off
    ProxyPass / http://127.0.0.1:3000/
    ProxyPassReverse / http://127.0.0.1:3000/
</VirtualHost>
```

## 备份和恢复

### 数据库备份
```bash
# PostgreSQL 备份
pg_dump -U wikijs -h localhost -d wiki > wiki_backup.sql

# PostgreSQL 恢复
psql -U wikijs -h localhost -d wiki < wiki_backup.sql

# 自动备份脚本
#!/bin/bash
DATE=$(date +%Y%m%d_%H%M%S)
pg_dump -U wikijs -h localhost -d wiki > /backup/wiki_$DATE.sql
find /backup -name "wiki_*.sql" -mtime +7 -delete
```

### 文件备份
```bash
# 备份上传文件
tar -czf wiki_uploads_$(date +%Y%m%d).tar.gz ./data/uploads

# 备份配置文件
cp config.yml config.yml.bak
```
