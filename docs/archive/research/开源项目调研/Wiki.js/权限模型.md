# Wiki.js 权限模型文档

## 概述

Wiki.js 实现了全面的权限系统，提供细粒度的访问控制。权限系统支持基于角色的访问控制（RBAC）和页面级别的规则。

## 权限系统架构

### 权限类型

Wiki.js 权限系统包括以下几种权限类型：

#### 1. 全局权限
系统范围的访问控制，适用于整个 wiki：

```javascript
{
  "global": {
    "manage:system": false,
    "manage:navigation": false,
    "manage:theme": false,
    "manage:users": false
  }
}
```

**关键全局权限**：
- `manage:system`: 系统管理权限
- `manage:navigation`: 导航管理权限
- `manage:theme`: 主题管理权限
- `manage:users`: 用户管理权限
- `write:users`: 创建新用户权限

#### 2. 内容权限
页面和内容相关的权限：

```javascript
{
  "content": {
    "read:pages": true,
    "write:pages": false,
    "delete:pages": false,
    "publish:pages": false
  }
}
```

**关键内容权限**：
- `read:pages`: 读取页面内容
- `write:pages`: 创建和编辑页面
- `delete:pages`: 删除页面
- `publish:pages`: 发布页面更改

#### 3. 功能权限
特定功能的访问权限：

```javascript
{
  "features": {
    "read:comments": true,
    "write:comments": true,
    "read:assets": true,
    "write:assets": false
  }
}
```

**关键功能权限**：
- `read:comments`: 查看评论
- `write:comments`: 发表评论
- `read:assets`: 访问资产
- `write:assets`: 上传和管理资产

#### 4. 页面规则
细粒度的页面级控制：

```javascript
{
  "pageRules": [
    {
      "path": "/internal/*",
      "match": "START",
      "roles": ["authenticated"],
      "rolesReject": ["guest"],
      "permissions": {
        "read:pages": true,
        "write:pages": false
      }
    }
  ]
}
```

## 角色和组

### 内置角色

Wiki.js 提供几个内置角色：

#### 1. 管理员
```javascript
{
  "name": "Administrator",
  "permissions": {
    "manage:system": true,
    "manage:navigation": true,
    "manage:theme": true,
    "manage:users": true,
    "write:users": true,
    "read:pages": true,
    "write:pages": true,
    "delete:pages": true,
    "publish:pages": true,
    "read:comments": true,
    "write:comments": true,
    "read:assets": true,
    "write:assets": true
  }
}
```

#### 2. 来宾
```javascript
{
  "name": "Guest",
  "permissions": {
    "read:pages": true,
    "read:comments": true,
    "read:assets": true
  }
}
```

#### 3. 认证用户
```javascript
{
  "name": "Authenticated User",
  "permissions": {
    "read:pages": true,
    "write:pages": false,
    "read:comments": true,
    "write:comments": true,
    "read:assets": true,
    "write:assets": false
  }

```

### 自定义角色
创建具有特定权限组合的自定义角色：

```javascript
{
  "name": "Content Editor",
  "permissions": {
    "read:pages": true,
    "write:pages": true,
    "delete:pages": false,
    "read:assets": true,
    "write:assets": true

```

### 用户组
将用户组织成组以简化权限管理：

```javascript
{
  "name": "Editors",

  "permissions": {
    "write:pages": true,
    "write:assets": true
  }
}
```

## 权限评估算法

### 权限检查流程

Wiki.js 使用复杂的算法来评估权限：

```
1. 检查用户认证状态
2. 获取用户角色和组成员身份
3. 收集所有适用的全局权限
4. 评估页面特定规则
5. 应用拒绝优先原则
6. 返回最终权限决策
```

### 权限继承
- **角色继承**: 用户继承角色的所有权限
- **组继承**: 用户继承所属组的权限
- **页面规则继承**: 页面规则可应用于子路径

### 权限优先级
1. **页面规则**: 最高优先级
2. **组权限**: 中等优先级
3. **角色权限**: 基础权限
4. **拒绝规则**: 拒绝始终覆盖允许

## 页面规则详解

### 匹配类型

#### EXACT 匹配
```javascript
{
  "path": "/internal/docs",
  "match": "EXACT",
  "permissions": {
    "read:pages": true
  }
}
```

#### START 匹配
```javascript
{
  "path": "/admin",
  "match": "START",
  "permissions": {
    "read:pages": true,
    "write:pages": true
  }
}
```

#### REGEX 匹配
```javascript
{
  "path": "/api/.*",
  "match": "REGEX",
  "permissions": {
    "read:pages": false
  }
}
```

### 角色过滤
```javascript
{
  "path": "/hr/*",
  "match": "START",
  "roles": ["hr-department"],
  "rolesReject": ["contractors"]
}
```

## 权限 API

### 检查权限
```http
POST /api/check-permissions
Content-Type: application/json

{
  "path": "/some-page",
  "operation": "write:pages"
}
```

### 获取用户权限
```http
GET /api/users/{id}/permissions
```

### 设置权限
```http
PUT /api/pages/{id}/permissions
Content-Type: application/json

{
  "rules": [
    {
      "path": "/private/*",
      "roles": ["admin"],
      "permissions": {
        "read:pages": true,
        "write:pages": true
      }
    }
  ]
}
```

## 安全最佳实践

### 1. 最小权限原则
- 仅授予用户完成任务所需的权限
- 定期审查和更新权限
- 删除不必要的权限

### 2. 分离职责
- 将管理权限分配给不同用户
- 使用审批流程进行关键操作
- 实施审计日志

### 3. 定期审计
```javascript
{
  "audit": {
    "enabled": true,
    "logLevel": "info",
    "retentionDays": 90
  }
}
```

### 4. 密码策略
```javascript
{
  "passwordPolicy": {
    "minLength": 12,
    "complexity": "high",
    "expirationDays": 90,
    "history": 5
  }
}
```

## 故障排除

### 常见权限问题

#### 用户无法访问页面
1. 检查用户角色和权限
2. 验证页面规则配置
3. 确认路径匹配规则
4. 检查认证状态

#### 权限不生效
1. 清除浏览器缓存
2. 检查权限缓存
3. 验证规则优先级
4. 检查认证提供者状态

#### 权限冲突
1. 审查所有适用规则
2. 检查拒绝规则
3. 验证页面规则优先级
4. 测试权限评估顺序
