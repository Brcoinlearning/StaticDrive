# Wiki.js 内容模型文档

## 概述

Wiki.js 采用灵活的内容模型，支持多种内容类型和编辑器。内容模型定义了如何创建、存储、管理和展示 wiki 页面。

## 页面结构

### 基本页面属性
每个 Wiki.js 页面包含以下核心属性：

```javascript
{
  "id": 123,
  "path": "my-page",
  "title": "My Page Title",
  "description": "Page description",
  "content": "Page content in markdown format",
  "locale": "en",
  "tags": ["tag1", "tag2"],
  "createdAt": "2025-01-01T00:00:00Z",
  "updatedAt": "2025-01-02T00:00:00Z",
  "authorId": 1,
  "editorId": 2
}
```

### 页面路径系统
- **层级结构**: 支持嵌套路径（如 `/docs/api/reference`）
- **路径唯一性**: 每个路径在语言内唯一
- **语言变体**: 同一路径可有多个语言版本
- **重定向**: 支持页面重定向设置

## 编辑器类型

Wiki.js 支持多种编辑器，每种编辑器使用不同的内容格式：

### 1. Markdown 编辑器
默认编辑器，使用标准 Markdown：
```markdown
# 标题
## 子标题

**粗体文本** 和 *斜体文本*

- 列表项 1
- 列表项 2

[链接](https://example.com)
![图片](/images/logo.png)
```

**特性**:
- 标准 Markdown 语法
- GitHub Flavored Markdown 扩展
- 代码块语法高亮
- 表格支持
- 自动链接识别

### 2. Code (MDX) 编辑器
使用 MDX (Markdown + JSX)：
```markdown
import { Chart } from './components/chart'

# 数据分析

<Chart data={salesData} />

详细的文本说明...
```

**特性**:
- Markdown + JavaScript 组件
- 动态内容渲染
- 可重用组件系统
- 数据可视化集成

### 3. Visual Editor (WYSIWYG)
所见即所得编辑器：
- 富文本编辑体验
- 格式化工具栏
- 实时预览
- 拖放布局

### 4. 其他编辑器
- **AsciiDoc**: 技术文档标准格式
- **PlantUML**: UML 图表
- **Mermaid**: 流程图和图表

## 内容渲染

### 渲染管道
1. **解析**: 根据编辑器类型解析内容
2. **处理**: 应用宏、插件和自定义处理
3. **渲染**: 生成最终 HTML 输出
4. **后处理**: 注入脚本、样式和交互元素

### 模板系统
Wiki.js 使用模板系统控制页面呈现：
```javascript
{
  "template": "custom-template",
  "variables": {
    "customField": "value",
    "layout": "sidebar"
  }
}
```

## 页面版本控制

### 版本历史
每个页面维护完整的版本历史：
```javascript
{
  "versions": [
    {
      "id": 1,
      "versionNumber": 1,
      "authorId": 1,
      "action": "created",
      "createdAt": "2025-01-01T00:00:00Z",
      "value": "Initial content"
    },
    {
      "id": 2,
      "versionNumber": 2,
      "authorId": 2,
      "action": "updated",
      "createdAt": "2025-01-02T00:00:00Z",
      "value": "Updated content"
    }
  ]
}
```

### 版本操作
- **查看历史**: 浏览所有版本
- **版本比较**: 查看版本间差异
- **回滚**: 恢复到先前版本
- **版本标签**: 标记重要版本

## 多语言支持

### 本地化模型
```javascript
{
  "locale": "en",
  "defaultLocale": "en",
  "translations": [
    {
      "locale": "fr",
      "path": "ma-page",
      "title": "Ma Page"
    }
  ]
}
```

### 翻译管理
- **语言检测**: 自动检测用户语言偏好
- **语言切换**: 页面间语言版本链接
- **翻译状态**: 跟踪翻译完成度
- **内容同步**: 跨语言内容同步

## 页面关系

### 链接系统
- **内部链接**: `[[页面名称]]` 或 `[文本](/路径)`
- **外部链接**: 标准 Markdown 链接
- **自动链接**: 自动识别 URL 并转换为链接
- **死链检查**: 自动检测和报告损坏链接

### 页面层级
```javascript
{
  "parent": "/docs",
  "children": [
    "/docs/api",
    "/docs/guide"
  ],
  "ancestors": ["/", "/docs"]
}
```

## 内容扩展

### 宏系统
Wiki.js 支持内容宏以扩展功能：
```markdown
# 页面标题

>>toc
>>video(path: 'my-video.mp4')
>>chart(type: 'bar', data: '{...}')
```

### 自定义组件
在 Code 编辑器中使用自定义组件：
```javascript
import CustomComponent from './components/custom'

# 标题

<CustomComponent prop="value" />
```

### 插件集成
- **搜索引擎插件**: 扩展搜索功能
- **分析插件**: 集成分析工具
- **社交媒体插件**: 社交分享功能

## 搜索和索引

### 全文搜索
- **内容索引**: 自动索引所有页面内容
- **元数据搜索**: 搜索标签、标题、作者
- **模糊匹配**: 支持模糊搜索
- **高亮显示**: 搜索结果中高亮匹配文本

### 搜索 API
```javascript
{
  "query": "search term",
  "locale": "en",
  "path": "/docs",
  "tags": ["api"],
  "limit": 20
}
```

## 导入和导出

### 支持的格式
- **Markdown**: 标准 .md 文件
- **HTML**: 富 HTML 内容
- **Word 文档**: .docx 文件
- **Confluence**: 从 Confluence 导入
- **MediaWiki**: 从 MediaWiki 导入

### 批量操作
- **批量导入**: 从文件系统导入
- **批量导出**: 导出为文件
- **格式转换**: 转换内容格式

## 性能优化

### 缓存策略
- **页面缓存**: 缓存渲染的页面
- **片段缓存**: 缓存可重用内容片段
- **智能失效**: 内容更新时自动失效缓存

### 内容分发
- **CDN 集成**: 静态资源 CDN 分发
- **边缘缓存**: 边缘节点缓存
- **预加载**: 预加载相关页面

1. **内容组织**: 使用清晰的路径结构
2. **一致格式**: 统一使用一种编辑器
3. **版本管理**: 定期创建重要版本标签
4. **翻译策略**: 规划多语言内容策略
5. **性能考虑**: 优化大型页面加载
6. **备份策略**: 定期备份内容
7. **搜索优化**: 使用适当的标题和标签

- 实施严格的层级结构

- 细粒度的权限控制
- 全面的审计日志

### 技术文档
- 使用技术文档专用编辑器
- 实施版本控制工作流
- 集成 API 文档工具
