# PocketBase 部署说明文档

## 概述

PocketBase 是一个开源后端解决方案，可以轻松部署到各种环境。本文档涵盖了从本地开发到生产环境的完整部署指南。

---

## 1. 系统要求

### 最低要求

- **操作系统**: Linux、macOS、Windows
- **架构**: x86_64、ARM64、ARMv7
- **内存**: 最低 64MB（推荐 256MB+）
- **磁盘空间**: 最低 100MB（根据数据增长）
- **网络**: 可选（如需远程访问）

### 支持的数据库

- **SQLite**（默认，嵌入式）
- 支持扩展其他数据库（需要自定义开发）

## 2. 本地开发部署

### 下载和安装

#### 方式 1: 直接下载

1. 访问 [PocketBase Releases](https://github.com/pocketbase/pocketbase/releases)
2. 下载适合你系统的二进制文件
3. 解压到目标目录

```bash
# Linux/macOS
wget https://github.com/pocketbase/pocketbase/releases/download/vX.X.X/pocketbase_X.X_X_linux_amd64.zip
unzip pocketbase_X.X_X_linux_amd64.zip

# Windows
# 下载并解压 .zip 文件
```

#### 方式 2: 使用包管理器

**Homebrew (macOS/Linux)**:
```bash
brew install pocketbase
```

**Scoop (Windows)**:
```bash
scoop bucket add extras
scoop install pocketbase
```

### 启动服务器

```bash
# 基本启动
./pocketbase serve

# 指定端口
./pocketbase serve --http=0.0.0.0:8090

# 指定数据目录
./pocketbase serve --dataDir=/path/to/data

# 开发模式（启用日志和自动重启）
./pocketbase serve --dev
```

### 访问管理面板

启动后访问：
- **管理面板**: `http://localhost:8090/_/`
- **API 端点**: `http://localhost:8090/api/`

首次访问需要创建管理员账户。

## 3. 生产环境部署

### 3.1 Linux 服务器部署

#### 使用 systemd 服务

创建服务文件 `/etc/systemd/system/pocketbase.service`:

```ini
[Unit]
Description=PocketBase
After=network.target

[Service]
Type=simple
User=pocketbase
Group=pocketbase
WorkingDirectory=/opt/pocketbase
ExecStart=/opt/pocketbase/pocketbase serve --http=0.0.0.0:8090
Restart=always
RestartSec=5s
StandardOutput=journal
StandardError=journal
SyslogIdentifier=pocketbase

# 安全设置
NoNewPrivileges=true
PrivateTmp=true

# 资源限制
LimitNOFILE=65535

[Install]
WantedBy=multi-user.target
```

创建用户和目录：
```bash
# 创建用户
sudo useradd -r -s /bin/false pocketbase

# 创建目录
sudo mkdir -p /opt/pocketbase
sudo chown pocketbase:pocketbase /opt/pocketbase

# 复制文件
sudo cp pocketbase /opt/pocketbase/
sudo chown pocketbase:pocketbase /opt/pocketbase/pocketbase
sudo chmod +x /opt/pocketbase/pocketbase
```

启用和启动服务：
```bash
# 重载 systemd 配置
sudo systemctl daemon-reload

# 启用服务（开机自启）
sudo systemctl enable pocketbase

# 启动服务
sudo systemctl start pocketbase

# 查看状态
sudo systemctl status pocketbase

# 查看日志
sudo journalctl -u pocketbase -f
```

#### 使用 Docker

**Dockerfile**:
```dockerfile
FROM alpine:latest

RUN apk add --no-cache ca-certificates

WORKDIR /pb

# 复制 PocketBase 二进制文件
COPY pocketbase /pb/pocketbase
COPY --chown=nonroot:nonroot --from=builder /pb/pb_data /pb/pb_data
COPY --chown=nonroot:nonroot --from=builder /pb/pb_public /pb/pb_public

# 暴露端口
EXPOSE 8090

# 设置用户
USER nonroot

# 启动命令
CMD ["/pb/pocketbase", "serve", "--http", "0.0.0.0:8090"]
```

**docker-compose.yml**:
```yaml
version: '3.8'

services:
  pocketbase:
    image: ghcr.io/muchobien/pocketbase:latest
    container_name: pocketbase
    restart: unless-stopped
    ports:
      - "8090:8090"
    volumes:
      - ./pb_data:/pb/pb_data
      - ./pb_public:/pb/pb_public
    environment:
      - TZ=Asia/Shanghai
    healthcheck:
      test: ["CMD", "wget", "-q", "--spider", "http://localhost:8090/api/health"]
      interval: 1m
      timeout: 10s
      retries: 3
      start_period: 10s
```

运行 Docker：
```bash
# 构建镜像
docker build -t my-pocketbase .

# 使用 docker-compose
docker-compose up -d

# 查看日志
docker-compose logs -f pocketbase

# 停止服务
docker-compose down
```

## 4. 反向代理配置

### Nginx

**配置文件** (`/etc/nginx/sites-available/pocketbase`):
```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://localhost:8090;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # 上传文件大小限制
        client_max_body_size 100M;
    }
}
```

启用配置：
```bash
sudo ln -s /etc/nginx/sites-available/pocketbase /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

### Caddy

**Caddyfile**:
```
your-domain.com {
    reverse_proxy localhost:8090

    # 上传文件大小限制
    request_body {
        max_size 100MB
    }
}
```

### Apache

**配置文件**:
```apache
<VirtualHost *:80>
    ServerName your-domain.com

    ProxyPreserveHost On
    ProxyRequests Off

    ProxyPass / http://localhost:8090/
    ProxyPassReverse / http://localhost:8090/

    # 上传文件大小限制
    LimitRequestBody 104857600
</VirtualHost>
```

## 6. 环境变量配置

### 常用环境变量

```bash
# 应用配置
export PB_ENCRYPTION_KEY="your-32-char-encryption-key"

export PB_SMTP_USERNAME="your-email@gmail.com"
export PB_SMTP_PASSWORD="your-app-password"
```

## 7. 数据备份和恢复

### 备份

#### 手动备份

```bash
# 停止服务
./pocketbase stop

# 备份数据目录
tar -czf pb_data_backup_$(date +%Y%m%d).tar.gz pb_data/

# 重启服务
./pocketbase serve
```

#### 自动备份脚本

```bash
#!/bin/bash
# backup.sh

BACKUP_DIR="/backups/pocketbase"
DATA_DIR="/opt/pocketbase/pb_data"
TIMESTAMP=$(date +%Y%m%d_%H%M%S)

mkdir -p $BACKUP_DIR

# 停止服务
systemctl stop pocketbase

# 备份
tar -czf $BACKUP_DIR/pb_data_$TIMESTAMP.tar.gz -C $DATA_DIR .

# 重启服务
systemctl start pocketbase

# 删除 7 天前的备份
find $BACKUP_DIR -name "pb_data_*.tar.gz" -mtime +7 -delete
```

#### 定时备份

添加到 crontab：
```bash
# 每天凌晨 2 点备份
0 2 * * * /opt/pocketbase/backup.sh
```

### 恢复

```bash
# 停止服务
systemctl stop pocketbase

# 备份当前数据
mv pb_data pb_data_old

# 恢复备份
tar -xzf pb_data_backup_20231201.tar.gz -C /

# 重启服务
systemctl start pocketbase
```

## 10. 安全最佳实践

### 1. 防火墙配置

```bash
# UFW
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw allow 22/tcp
sudo ufw enable

# 仅允许特定 IP 访问管理面板
sudo ufw allow from YOUR_IP to any port 8090
```

### 2. 更新加密密钥

生成安全的加密密钥：
```bash
openssl rand -base64 32
```

### 3. 限制管理面板访问

```nginx
# Nginx 配置
location /_/ {
    allow YOUR_IP;
    deny all;
    proxy_pass http://localhost:8090;
}
```

### 4. 使用环境变量存储敏感信息

不要在代码中硬编码密钥和密码。

### 5. 定期更新

```bash
# 检查更新
curl -s https://pockethost.io/api/latest-version

# 下载新版本
wget https://github.com/pocketbase/pocketbase/releases/download/vX.X.X/pocketbase_X.X_X_linux_amd64.zip

# 备份并替换
./backup.sh
unzip pocketbase_X.X_X_linux_amd64.zip
systemctl restart pocketbase
```

### 6. API 速率限制

在自定义扩展中添加速率限制：

```go
e.Router.GET("/api/limited", handler, middlewares.RateLimiter(100, time.Hour))
```

## 11. 故障排除

### 常见问题

1. **服务无法启动**
```bash
# 检查权限
ls -la pb_data pb_public

# 检查端口占用
sudo lsof -i :8090

# 检查日志
journalctl -u pocketbase -n 50
```

2. **数据库锁定**
```bash
# 检查锁文件
ls -la pb_data/*.db-*-lock

# 删除锁文件（确保没有其他进程运行）
rm pb_data/*.db-*-lock
```

3. **内存不足**
```bash
# 检查内存使用
free -h

# 增加交换空间
sudo dd if=/dev/zero of=/swapfile bs=1M count=1024
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
```

4. **上传失败**
- 检查文件大小限制
- 检查磁盘空间
- 验证文件权限

## 13. 部署工具

### PocketHost

[PocketHost](https://pockethost.io) 是 PocketBase 的官方托管服务：

- 免费层可用
- 自动 SSL
- 一键部署
- 自动备份
- 简单的域名管理

### 部署脚本

```bash
#!/bin/bash
# deploy.sh

VERSION="0.22.0"
DOWNLOAD_URL="https://github.com/pocketbase/pocketbase/releases/download/v${VERSION}/pocketbase_${VERSION}_linux_amd64.zip"

# 下载
wget $DOWNLOAD_URL -O pocketbase.zip

# 备份
./backup.sh

# 停止服务
systemctl stop pocketbase

# 更新
unzip -o pocketbase.zip -d /opt/pocketbase/
chmod +x /opt/pocketbase/pocketbase

# 启动服务
systemctl start pocketbase

# 清理
rm pocketbase.zip

echo "部署完成！"
```
