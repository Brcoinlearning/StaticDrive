# Appwrite 自托管架构说明

## 概述

Appwrite 是一个完整的后端服务器解决方案，采用微服务架构设计，专为自托管环境优化。它将常用的后端服务封装成统一的API，包括数据库、存储、用户认证等功能。

## 核心架构组件

### 1. 微服务架构

Appwrite 采用以下微服务架构：

```
┌─────────────────────────────────────────────────────────┐
│                    Appwrite Console                      │
│                  (管理界面 - React)                       │
└─────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────┐
│                     API Gateway                          │
│                  (Nginx 反向代理)                         │
└─────────────────────────────────────────────────────────┘
                            │
        ┌──────────────────┼──────────────────┐
        │                  │                  │
        ▼                  ▼                  ▼
┌──────────────┐  ┌──────────────┐  ┌──────────────┐
│   Appwrite   │  │   Appwrite   │  │   Appwrite   │
│    API       │  │   Database   │  │   Storage    │
│  (Main API)  │  │  (MariaDB)   │  │   Service    │
└──────────────┘  └──────────────┘  └──────────────┘
        │                  │                  │
        └──────────────────┼──────────────────┘
                           │
        ┌──────────────────┼──────────────────┐
        │                  │                  │
        ▼                  ▼                  ▼
┌──────────────┐  ┌──────────────┐  ┌──────────────┐
│   Functions  │  │    Realtime  │  │    Workers   │
│   Service    │  │   Service    │  │   Service    │
└──────────────┘  └──────────────┘  └──────────────┘
```

### 2. 主要服务组件

#### API 服务 (appwrite/api)
- **职责**: 处理所有HTTP请求的核心API服务
- **技术栈**: PHP + Symfony Framework
- **端口**: 80 (HTTP)
- **功能**:
  - 路由和请求分发
  - 中间件处理
  - 响应格式化
  - 错误处理

#### 数据库服务 (appwrite/database)
- **职责**: 数据持久化和查询
- **技术栈**: MariaDB
- **端口**: 3306
- **功能**:
  - 用户数据存储
  - 文档集合管理
  - 关系和索引
  - 事务处理

#### 存储服务 (appwrite/storage)
- **职责**: 文件上传和管理
- **技术栈**: PHP + 本地文件系统
- **端口**: 无独立端口
- **功能**:
  - 文件上传/下载
  - 文件转换
  - CDN 集成
  - 存储配额管理

#### Functions 服务 (appwrite/functions)
- **职责**: 执行自定义函数
-技术栈**: Docker + Runtime
- **端口**: 动态分配
- **功能**:
  - 代码执行
  - 环境隔离
  - 资源限制
  - 日志收集

#### Realtime 服务 (appwrite/realtime)
- **职责**: 实时数据推送
- **技术栈**: PHP + WebSockets
- **端口**: 80
- **功能**:
  - WebSocket 连接管理
  - 事件订阅
  - 消息广播
  - 房间管理

#### Workers 服务 (appwrite/worker)
- **职责**: 后台任务处理
- **技术栈**: PHP + Redis Queue
- **端口**: 无独立端口
- **功能**:
  - 异步任务执行
  - 定时任务
  - 邮件发送
  - 数据清理

## Docker 部署架构

### Docker Compose 配置

Appwrite 使用 Docker Compose 进行容器编排，主要容器包括：

```yaml
version: '3.8'

services:
  # Appwrite 主服务
  appwrite:
    image: appwrite/appwrite:latest
    ports:
      - "80:80"
      - "443:443"
    environment:
      - _APP_ENV=production
      - _APP_OPENSSL_KEY_V1=your-key
    volumes:
      - appwrite-data:/storage
    depends_on:
      - mariadb
      - redis

  # MariaDB 数据库
  mariadb:
    image: mariadb:10.6
    ports:
      - "3306:3306"
    environment:
      - MYSQL_ROOT_PASSWORD=password
      - MYSQL_DATABASE=appwrite
    volumes:
      - mariadb-data:/var/lib/mysql

  # Redis 缓存和队列
  redis:
    image: redis:6.2
    ports:
      - "6379:6379"
    volumes:
      - redis-data:/data

  # 实时服务
  realtime:
    image: appwrite/realtime:latest
    ports:
      - "80:80"
    depends_on:
      - redis

volumes:
  appwrite-data:
  mariadb-data:
  redis-data:
```

### 容器说明

1. **appwrite/appwrite**
   - 核心API服务器
   - 提供RESTful API
   - 处理所有业务逻辑

2. **mariadb**
   - 关系型数据库
   - 存储结构化数据
   - 支持事务和关系

3. **redis**
   - 内存缓存
   - 任务队列
   - 会话存储

4. **appwrite/realtime**
   - WebSocket服务器
   - 实时数据推送
   - 事件订阅

## 数据流架构

### 请求处理流程

```
用户请求
    │
    ▼
Nginx (反向代理)
    │
    ▼
API Gateway (路由)
    │
    ├─→ Auth Service (认证)
    ├─→ Database Service (数据)
    ├─→ Storage Service (文件)
    ├─→ Functions Service (函数)
    └─→ Realtime Service (实时)
    │
    ▼
响应返回
```

### 数据存储流程

```
上传请求
    │
    ▼
Storage API
    │
    ├─→ 验证权限
    ├─→ 文件处理
    ├─→ 存储文件
    └─→ 更新元数据
    │
    ▼
返回文件ID
```

```

    │

    ▼

    │

    ▼
```

## 总结

Appwrite 的自托管架构具有以下特点：

- **微服务设计**: 模块化、可扩展
- **容器化部署**: 简化安装和维护
- **统一API**: 一致的开发体验
- **完整功能**: 涵盖常见后端需求
- **自托管友好**: 支持私有化部署

该架构适合需要数据主权、定制化需求或成本控制的项目使用。
