# Pingvin Share 架构概述

## 项目简介

Pingvin Share 是一个自托管的文件分享平台，作为 WeTransfer 的开源替代方案。项目使用现代化的全栈架构，提供轻量级且美观的文件分享体验。

**项目状态：已归档（2025-06-29）**
- 作者决定专注于 Pocket ID 项目
- 不再积极维护
- 欢迎社区 fork 并继续开发

## 数据模型

### 核心实体关系

```
User (用户)
├── Share[] (分享)
│   ├── File[] (文件)
│   ├── ShareRecipient[] (收件人)
│   ├── ShareSecurity (安全设置)
│   └── ReverseShare (反向分享)
├── RefreshToken[] (刷新令牌)
├── LoginToken[] (登录令牌)
├── ResetPasswordToken (重置密码令牌)
└── OAuthUser[] (OAuth 用户)
```

### Prisma Schema 关键模型

#### User 模型
```prisma
model User {
  id        String   @id @default(uuid())
  username  String   @unique
  email     String   @unique
  password  String?
  isAdmin   Boolean  @default(false)
  ldapDN    String?  @unique

  totpEnabled   Boolean @default(false)
  totpSecret    String?

  shares        Share[]
  reverseShares ReverseShare[]
}
```

#### Share 模型
```prisma
model Share {
  id            String   @id @default(uuid())
  name          String?
  description   String?
  expiration    DateTime
  views         Int      @default(0)
  uploadLocked  Boolean  @default(false)

  creatorId     String?
  creator       User?    @relation(...)

  security      ShareSecurity?
  recipients    ShareRecipient[]
  files         File[]

  storageProvider String @default("LOCAL")
}
```

#### File 模型
```prisma
model File {
  id        String   @id @default(uuid())
  name      String
  size      String

  shareId   String
  share     Share    @relation(...)
}
```

#### ShareSecurity 模型
```prisma
model ShareSecurity {
  id       String   @id @default(uuid())
  password String?
  maxViews  Int?

  shareId  String?  @unique
  share    Share?   @relation(...)
}
```

## 文件处理流程

### 上传流程

```
┌─────────────────────────────────────────────────────────────┐
│                     客户端上传                               │
│  1. 文件分块 (chunkSize: 10MB)                              │
│  2. 并发上传 (p-limit)                                      │
│  3. 进度跟踪                                                │
└────────────────────┬────────────────────────────────────────┘
                     │
┌────────────────────▼────────────────────────────────────────┐
│                     后端处理                                 │
│  1. 验证分享权限                                             │
│  2. 检查文件大小限制                                         │
│  3. ClamAV 病毒扫描（可选）                                  │
│  4. 保存到存储（本地/S3）                                     │
│  5. 更新数据库记录                                           │
└─────────────────────────────────────────────────────────────┘
```

### 下载流程

```
┌─────────────────────────────────────────────────────────────┐
│                     访问分享                                 │
│  1. 验证分享ID                                               │
│  2. 检查过期时间                                             │
│  3. 验证密码（如设置）                                        │
│  4. 检查访问次数限制                                         │
└────────────────────┬────────────────────────────────────────┘
                     │
┌────────────────────▼────────────────────────────────────────┐
│                     生成下载                                 │
│  1. 收集所有文件                                             │
│  2. 创建 ZIP（压缩级别可配置）                                │
│  3. 流式传输给客户端                                         │
│  4. 更新访问计数                                             │
└─────────────────────────────────────────────────────────────┘
```

## 总结

Pingvin Share 采用现代化的全栈架构，具有以下特点：

**优势：**
- 清晰的模块化设计
- 灵活的配置系统
- 多种认证方式
- 良好的安全实践
- 易于部署（Docker）

**劣势：**
- 项目已归档
- 文档不够完善
- 某些功能缺乏深度
- 社区活跃度下降

**适用场景：**
- 小型团队文件分享
- 临时文件传输
- 内部文件共享
- 学习和参考架构设计

**不适用场景：**
- 大规模生产环境
- 长期关键业务
- 需要持续维护的项目
