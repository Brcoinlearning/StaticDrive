# Pingvin Share 规则说明

## 概述

Pingvin Share 提供了灵活的规则配置系统，用于控制文件分享的各种限制和安全策略。由于项目已归档，以下规则基于最新版本（v1.13.0）的配置。

## 分享限制规则

### 1. 文件大小限制

```yaml
share:
  # 最大分享大小（字节）
  maxSize: "1000000000"  # 默认 1GB
```

**说明：**
- 控制单个分享的最大总大小
- 所有文件总和不能超过此限制
- 设置为 0 或留空表示无限制（仅受磁盘空间限制）
- 建议根据服务器存储容量合理设置

**示例值：**
- `1000000000` = 1GB
- `10000000000` = 10GB
- `0` = 无限制

### 2. 过期时间规则

```yaml
share:
  # 最大分享过期时间
  maxExpiration: 0 days
```

**说明：**
- 控制分享链接的最大有效期
- 设置为 0 表示允许永久分享
- 可使用不同时间单位：days, hours, minutes

**示例配置：**
```yaml
# 7天过期
maxExpiration: 7 days

# 24小时过期
maxExpiration: 24 hours

# 永不过期
maxExpiration: 0 days
```

**注意事项：**
- 永久分享可能占用大量存储空间
- 建议根据业务需求设置合理的过期时间
- 过期分享会自动清理

### 3. 访问次数限制

```yaml
security:
  # 最大访问次数
  maxViews: Int?
```

**说明：**
- 限制分享链接可以被访问的次数
- 达到限制后分享将无法访问
- 留空或设置为 null 表示无限制

**使用场景：**
- 临时文件分享（设置较低的访问次数）
- 敏感文件分享（限制访问次数）
- 一次性分享（设置为 1）

### 4. 分享ID长度规则

```yaml
share:
  # 分享ID长度
  shareIdLength: "8"
```

**说明：**
- 控制生成分享链接的ID长度
- 影响链接的安全性和可猜测性
- 建议不小于 8（低于8被认为不安全）

**安全性分析：**
- 长度 6：约 56 亿种组合（容易被暴力破解）
- 长度 8：约 2.8 万亿种组合（基本安全）
- 长度 10：约 1400 万亿种组合（高度安全）

**ID空间计算：**
- 使用字符集：大小写字母 + 数字（62个字符）
- 组合数 = 62^长度

## 认证规则

### 1. 注册控制

```yaml
share:
  # 允许新用户注册
  allowRegistration: "true"
```

**规则说明：**
- 控制是否开放用户注册
- 关闭后仅管理员可创建用户
- 公共服务建议保持开启

### 2. 未认证用户规则

```yaml
share:
  # 允许未认证用户创建分享
  allowUnauthenticatedShares: "false"
```

**规则说明：**
- 控制是否允许未登录用户创建分享
- 关闭后必须登录才能创建分享
- 影响反向分享功能

**安全建议：**
- 私有部署：可设置为 false
- 公共服务：建议设置为 true（但要配合其他限制）

### 3. 会话持续时间

```yaml
general:
  # 用户会话持续时间
  sessionDuration: 3 months
```

**说明：**
- 控制用户登录后的有效期
- 超时后需要重新登录
- 可使用不同时间单位

**示例配置：**
```yaml
# 1天
sessionDuration: 1 day

# 1周
sessionDuration: 1 week

# 3个月（默认）
sessionDuration: 3 months
```

## 密码规则

### 1. 分享密码

```yaml
security:
  # 分享密码（可选）
  password: String?
```

**规则说明：**
- 为分享添加密码保护
- 访问者需要输入正确密码才能访问
- 可选配置，不强制要求

**最佳实践：**
- 敏感文件务必设置密码
- 密码应足够复杂
- 定期更换重要分享的密码

### 2. 用户密码

用户密码规则：
- 最小长度：由前端验证决定
- 密码强度：建议使用强密码
- 密码重置：通过邮件重置

**TOTP（双因素认证）：**
```yaml
user:
  # TOTP配置
  totpEnabled: false
  totpVerified: false
  totpSecret: String?
```

## OAuth 认证规则

```yaml
oauth:
  # 允许通过OAuth注册
  allowRegistration: "true"

  # OAuth登录时忽略TOTP
  ignoreTotp: "true"

  # 禁用密码登录
  disablePassword: "false"
```

**规则说明：**
- `allowRegistration`：允许通过OAuth自动注册新用户
- `ignoreTotp`：OAuth用户可跳过TOTP验证
- `disablePassword`：完全禁用密码登录（仅OAuth）

**安全警告：**
- 禁用密码登录前确保OAuth配置正确
- 避免被锁定在系统外

## LDAP 认证规则

```yaml
ldap:
  # 启用LDAP认证
  enabled: "false"

  # LDAP服务器配置
  url: ""
  bindDn: ""
  bindPassword: ""
  searchBase: ""
  searchQuery: ""

  # 管理员组
  adminGroups: ""

  # 字段映射
  fieldNameMemberOf: memberOf
  fieldNameEmail: userPrincipalName
```

**规则说明：**
- 集成企业LDAP/AD认证
- 支持管理员组权限控制
- 用户信息从LDAP同步

**配置要点：**
- `searchQuery` 中的 `%username%` 会被替换为用户输入
- `adminGroups` 中的用户自动获得管理员权限
- `ldapDN` 存储用户的LDAP DN

## 邮件规则

### 1. SMTP 配置规则

```yaml
smtp:
  # 启用SMTP
  enabled: "false"

  # 证书验证
  allowUnauthorizedCertificates: "false"

  # 连接配置
  host: ""
  port: "0"
  email: ""
  username: ""
  password: ""
```

**使用规则：**
- 只有在 `enabled: true` 时才发送邮件
- 发件邮件功能需要SMTP配置
- 生产环境应配置有效SMTP

### 2. 邮件发送限制

```yaml
email:
  # 启用分享邮件通知
  enableShareEmailRecipients: "false"
```

**规则说明：**
- 控制是否启用邮件通知功能
- 依赖 SMTP 配置
- 影响分享收件人和反向分享通知

## 反向分享规则

```yaml
reverseShare:
  # 令牌
  token: uuid

  # 分享过期时间
  shareExpiration: DateTime

  # 最大分享大小
  maxShareSize: String

  # 剩余使用次数
  remainingUses: Int

  # 公开访问
  publicAccess: true

  # 简化模式
  simplified: false
```

**规则说明：**

1. **remainingUses** - 使用次数限制
   - 控制反向链接可以被使用的次数
   - 每次使用减1，减到0后失效
   - 设置为负数表示无限制

2. **maxShareSize** - 单次分享大小限制
   - 限制通过此链接创建的单个分享大小
   - 防止滥用存储空间

3. **publicAccess** - 公开访问控制
   - true：任何人都可以使用此链接
   - false：需要登录才能使用

4. **simplified** - 简化模式
   - true：简化的上传界面
   - false：完整的分享配置界面

## 存储规则

### 1. 上传分块规则

```yaml
share:
  # 分块大小
  chunkSize: "10000000"  # 10MB
```

**规则说明：**
- 控制文件上传的分块大小
- 影响上传成功率和速度

**推荐配置：**
- 稳定网络：20-50MB
- 不稳定网络：5-10MB
- 移动网络：2-5MB

### 2. 压缩规则

```yaml
share:
  # ZIP压缩级别
  zipCompressionLevel: "9"
```

**规则说明：**
- 范围：0-9
- 0 = 无压缩（最快）
- 9 = 最大压缩（最慢）
- 默认：9

**性能权衡：**
- 高压缩：节省带宽但消耗CPU
- 低压缩：快速传输但文件较大

## ClamAV 扫描规则

集成 ClamAV 进行病毒扫描：

```yaml
# ClamAV 配置
clamav:
  enabled: false
  # 扫描所有上传文件
  # 阻止恶意文件
```

**扫描规则：**
- 上传时自动扫描
- 发现病毒立即阻止
- 记录扫描结果

## 安全建议

1. **生产环境配置**
   - shareIdLength >= 10
   - 启用密码保护
   - 设置合理的过期时间
   - 限制文件大小
   - 启用 ClamAV 扫描

2. **公共访问规则**
   - 关闭 allowUnauthenticatedShares
   - 启用邮件验证
   - 限制分享ID长度
   - 监控存储使用

3. **企业部署规则**
   - 启用 LDAP 认证
   - 配置管理员组
   - 设置会话超时
   - 启用审计日志

## 规则优先级

配置规则的优先级顺序：

1. **环境变量** - 最高优先级
2. **配置文件 (config.yaml)** - 中等优先级
3. **UI 配置** - 可被管理员覆盖
4. **默认值** - 最低优先级

## 注意事项

1. **项目已归档**
   - 不再接收新功能
   - 安全漏洞可能不会修复
   - 建议考虑替代方案

2. **数据迁移**
   - 规则变更可能影响现有分享
   - 建议在测试环境先验证
   - 备份重要数据

3. **性能考虑**
   - 过度限制可能影响用户体验
   - 压缩级别影响CPU使用
   - 分块大小影响上传速度
