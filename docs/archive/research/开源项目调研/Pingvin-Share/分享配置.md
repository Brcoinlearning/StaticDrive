# Pingvin Share 分享配置

## 项目概述

Pingvin Share 是一个自托管的文件分享平台，作为 WeTransfer 的替代方案。该项目已于 2025 年 6 月 29 日被归档，作者 stonith404 表示将专注于另一个项目 Pocket ID 的开发。

## 基本分享功能

### 核心特性

- **通过链接分享文件** - 用户可以生成分享链接与他人共享文件
- **无限文件大小** - 仅受磁盘空间限制
- **设置过期日期** - 可为分享设置过期时间
- **安全选项** - 支持访问次数限制和密码保护
- **邮件通知** - 可向收件人发送邮件通知
- **反向分享** - 允许他人通过您的链接创建分享

### 分享配置选项

#### 1. 基本配置

```yaml
share:
  # 是否允许注册
  allowRegistration: "true"

  # 是否允许未认证用户创建分享
  allowUnauthenticatedShares: "false"

  # 分享ID长度（默认8，低于8被认为不安全）
  shareIdLength: "8"

  # 最大分享大小（字节）
  maxSize: "1000000000"  # 1GB
```

#### 2. 过期时间配置

```yaml
share:
  # 最大分享过期时间
  # 设置为 0 表示允许无限期
  # 可用单位：days, hours, minutes
  maxExpiration: 0 days  # 默认无限制
```

#### 3. 文件处理配置

```yaml
share:
  # ZIP压缩级别 (0-9)
  # 0 = 无压缩，9 = 最大压缩
  zipCompressionLevel: "9"

  # 上传分块大小（字节）
  # 较小的分块适合不稳定的连接
  # 较大的分块适合稳定的连接
  chunkSize: "10000000"  # 10MB

  # 自动打开分享创建模态框
  autoOpenShareModal: "false"
```

## 分享类型

### 1. 普通分享

- 创建者上传文件并生成分享链接
- 可配置访问密码和访问次数限制
- 可设置过期时间
- 支持文件描述

### 2. 反向分享 (Reverse Share)

允许其他用户通过您的链接创建分享：

```yaml
# 反向分享配置
share:
  # 反向分享令牌
  token: uuid

  # 分享过期时间
  shareExpiration: DateTime

  # 最大分享大小
  maxShareSize: String

  # 发送邮件通知
  sendEmailNotification: Boolean

  # 剩余使用次数
  remainingUses: Int

  # 简化模式
  simplified: false

  # 公开访问
  publicAccess: true
```

## 安全配置

### 1. 密码保护

```yaml
security:
  # 分享密码（可选）
  password: String?

  # 最大访问次数
  maxViews: Int?
```

### 2. 访问控制

```yaml
general:
  # 应用名称
  appName: Pingvin Share

  # 应用URL
  appUrl: http://localhost:3000

  # 安全Cookie标志
  # 启用后HTTP访问将无法工作
  secureCookies: "false"

  # 显示首页
  showHomePage: "true"

  # 会话持续时间（默认3个月）
  sessionDuration: 3 months
```

## 邮件配置

### SMTP 设置

```yaml
smtp:
  # 启用SMTP
  enabled: "false"

  # 允许自签名证书
  allowUnauthorizedCertificates: "false"

  # SMTP服务器
  host: ""
  port: "0"

  # 发件邮箱
  email: ""

  # SMTP认证
  username: ""
  password: ""
```

### 邮件模板

```yaml
email:
  # 启用邮件分享
  enableShareEmailRecipients: "false"

  # 分享通知主题
  shareRecipientsSubject: Files shared with you

  # 分享通知内容（支持变量）
  shareRecipientsMessage: |-
    Hey!
    {creator} ({creatorEmail}) shared some files with you.
    Link: {shareUrl}
    Expires: {expires}
    Note: {desc}
```

支持的邮件变量：
- `{creator}` - 创建者用户名
- `{creatorEmail}` - 创建者邮箱
- `{shareUrl}` - 分享链接
- `{desc}` - 分享描述
- `{expires}` - 过期日期

## Docker 配置

### 基础 Docker Compose

```yaml
services:
  pingvin-share:
    image: stonith404/pingvin-share
    restart: unless-stopped
    ports:
      - 3000:3000
    environment:
      # 如果有反向代理则设置为 true
      - TRUST_PROXY=false
    volumes:
      - "./data:/opt/app/backend/data"
      - "./data/images:/opt/app/frontend/public/img"

      # - "./config.yaml:/opt/app/config.yaml"
```

## 使用建议

1. **生产环境部署**
   - 修改默认密码
   - 配置 HTTPS
   - 启用安全Cookie
   - 配置 SMTP 服务

2. **安全建议**
   - shareIdLength 不低于 8
   - 启用密码保护敏感分享
   - 设置合理的过期时间
   - 配置 ClamAV 扫描上传文件

3. **性能优化**
   - 根据网络状况调整 chunkSize
   - 根据需求调整 zipCompressionLevel
   - 多实例部署时启用 Redis 缓存
