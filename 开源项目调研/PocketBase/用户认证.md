# PocketBase 用户认证文档

## 概述

PocketBase 提供了完整的用户认证系统，支持多种认证方式，包括密码登录、OAuth2、OTP（一次性密码）等。认证系统是可配置的，可以根据需求定制。

---

## 用户集合 (Users Collection)

PocketBase 使用特殊的"用户集合"来管理用户身份。用户集合可以具有以下特殊功能：

### 用户集合功能

1. **身份认证** - 支持多种登录方式
2. **邮箱验证** - 发送验证邮件
3. **密码重置** - 通过邮件重置密码
4. **OAuth2 集成** - 第三方登录
5. **OTP 认证** - 一次性密码登录

### 默认用户集合字段

创建用户集合时，通常包含以下字段：

- **email** - 电子邮件地址 (email 类型)
- **name** - 用户名 (text 类型)
- **avatar** - 头像 (file 类型)
- **verified** - 邮箱验证状态 (bool 类型)
- **emailVisibility** - 邮箱可见性 (bool 类型)

## 密码认证

### 用户注册

```javascript
// 创建新用户
const record = await pb.collection('users').create({
  email: 'user@example.com',
  password: 'secure_password_123',
  passwordConfirm: 'secure_password_123',
  name: 'John Doe'
});

console.log(record);
```

### 密码登录

```javascript
// 使用密码登录
const authData = await pb.collection('users').authWithPassword(
  'user@example.com',
  'secure_password_123'
);

// authData 包含:
// - token: 用于认证的令牌
// - record: 用户记录
// - cookie: (如果使用 cookie 认证)

console.log(authData.token);
console.log(authData.record.email);

// PocketBase 会自动保存 token 和记录
console.log(pb.authStore.token);
console.log(pb.authStore.model);
```

### 检查登录状态

```javascript
// 检查用户是否已登录
if (pb.authStore.isValid) {
  console.log('用户已登录:', pb.authStore.model.email);
} else {
  console.log('未登录');
}

// 获取当前用户
const currentUser = pb.authStore.model;
```

### 登出

```javascript
// 清除认证状态
pb.authStore.clear();

console.log('已登出');
```

## OAuth2 认证

PocketBase 支持多种 OAuth2 提供商：

- Google
- Facebook
- GitHub
- GitLab
- Discord
- Apple
- Microsoft
- Spotify
- Twitter/X
- VK
- Yandex
- Kakao

### 配置 OAuth2

1. 在管理面板中：**Settings > External Auth**
2. 添加提供商：
   - 输入 **Client ID**
   - 输入 **Client Secret**
   - 配置 **Redirect URL**
   - 启用提供商

### OAuth2 登录流程

#### 方式 1: 使用 JavaScript SDK

```javascript
import PocketBase from 'pocketbase';

const pb = new PocketBase('http://localhost:8090');

// 触发 OAuth2 登录
// 这会打开一个弹出窗口进行认证
const authData = await pb.collection('users').authWithOAuth2({
  provider: 'google',
  urlCallback: function(url) {
    // 在新窗口中打开 URL
    window.open(url, '_blank');
  }
});

// 或者使用重定向方式
pb.collection('users').authWithOAuth2({
  provider: 'google',
  // 如果创建一个新用户，可以提供默认值
  createData: {
    name: 'John Doe'
  }
});
```

#### 方式 2: 使用重定向流程

```javascript
// 1. 重定向到 OAuth2 提供商
const redirectUrl = 'http://localhost:8090/oauth2-redirect';

const authUrl = pb.buildOAuth2Url(
  'google',
  redirectUrl,
  // 可选的状态参数
  'random_state_string'
);

window.location.href = authUrl;

// 2. 在回调页面处理返回
// URL 会包含 code 和状态参数
```

### 自定义 OAuth2 配置

```javascript
// 自定义 OAuth2 参数
const authData = await pb.collection('users').authWithOAuth2(
  'google',
  'authorization_code_from_callback',
  'redirect_url_used',
  'code_verifier' // PKCE code verifier
);
```

## OTP 认证（一次性密码）

OTP 认证允许用户通过电子邮件或短信收到一次性登录代码，无需密码。

### 请求 OTP

```javascript
// 发送 OTP 到用户的邮箱
await pb.collection('users').requestOTP('user@example.com');

// 返回值
{
  "tokenId": "...",
  "email": "user@example.com"
}
```

### 使用 OTP 登录

```javascript
// 用户收到 OTP 后，使用它登录
const authData = await pb.collection('users').authWithOTP(
  'user@example.com',
  '123456' // 用户收到的 OTP 代码
);

console.log(authData.token, authData.record);
```

## 邮箱验证

### 启用邮箱验证

在集合设置中：
1. 编辑用户集合
2. 启用 **Require email verification**
3. 配置邮件模板

### 请求验证邮件

```javascript
// 发送验证邮件
await pb.collection('users').requestVerification('user@example.com');
```

### 确认邮箱验证

```javascript
// 用户点击邮件中的链接后，使用令牌验证
const result = await pb.collection('users').confirmVerification(
  'verification_token_from_email'
);

console.log(result);
```

### 检查验证状态

```javascript
// 获取用户信息
const user = await pb.collection('users').getOne('USER_ID');

if (user.verified) {
  console.log('邮箱已验证');
} else {
  console.log('邮箱未验证');
}
```

### 确认密码重置

```javascript

  'reset_token_from_email',
```

```javascript

  oldPassword: 'old_password',
  password: 'new_password',
  passwordConfirm: 'new_password'
});
```

## 认证中间件和规则

### API 规则

在集合设置中，可以配置 API 规则来控制访问：

```
# 允许所有用户读取
create = @request.auth.id != ""
list = @request.auth.id != ""
view = @request.auth.id != ""
update = @request.auth.id == id
delete = @request.auth.id == id
```

### 常见规则表达式

```javascript
// 仅允许已认证用户
@request.auth.id != ""

// 仅允许记录所有者
@request.auth.id == id

// 仅允许管理员
@request.auth.role == "admin"

// 基于字段的权限
@request.auth.id == author.id

// 组合条件
(@request.auth.id == id || @request.auth.role == "admin")
```

## 使用认证令牌

### Bearer Token

```bash
# 在请求头中发送令牌
curl http://localhost:8090/api/collections/posts/records \
  -H "Authorization: Bearer YOUR_TOKEN"
```

```javascript
// 手动设置令牌
pb.authStore.save(TOKEN, RECORD);

// 清除令牌
pb.authStore.clear();
```

### Cookie 认证

PocketBase 也支持基于 Cookie 的认证：

```javascript
// 启用 Cookie 存储
import { LocalAuthStorage } from 'pocketbase';

const pb = new PocketBase('http://localhost:8090', {
  authStorage: new LocalAuthStorage() // 使用 localStorage
});
```

## 多因素认证 (MFA)

PocketBase 支持 TOTP（基于时间的一次性密码）多因素认证：

### 设置 MFA

```javascript
// 为用户启用 MFA
const mfaSetup = await pb.collection('users').requestMFASetup();

// mfaSetup 包含:
// - secret: TOTP 密钥
// - qrCode: 二维码 URL

// 用户需要在认证应用中扫描二维码
```

### 验证 MFA

```javascript
// 使用 TOTP 代码登录
const authData = await pb.collection('users').authWithPassword(
  'user@example.com',
  'password123',
  {
    totpCode: '123456' // 来自认证应用的代码
  }
);
```

### 禁用 MFA

```javascript
// 禁用用户的 MFA
await pb.collection('users').disableMFA();
```

## 认证示例

### 完整登录流程

```javascript
import PocketBase from 'pocketbase';

const pb = new PocketBase('http://localhost:8090');

async function login(email, password) {
  try {
    const authData = await pb.collection('users').authWithPassword(
      email,
      password
    );

    // 登录成功
    console.log('欢迎!', authData.record.name);
    return authData;
  } catch (error) {
    console.error('登录失败:', error.message);
    throw error;
  }
}

async function register(email, password, name) {
  try {
    const record = await pb.collection('users').create({
      email,
      password,
      passwordConfirm: password,
      name
    });

    // 注册成功，可以自动登录
    console.log('注册成功!');
    return record;
  } catch (error) {
    console.error('注册失败:', error.message);
    throw error;
  }
}

async function logout() {
  pb.authStore.clear();
  console.log('已登出');
}
```

### React 认证 Hook

```jsx
import React, { createContext, useContext, useState } from 'react';
import PocketBase from 'pocketbase';

const pb = new PocketBase('http://localhost:8090');

const AuthContext = createContext();

export function AuthProvider({ children }) {
  const [user, setUser] = useState(pb.authStore.model);

  pb.authStore.onChange(() => {
    setUser(pb.authStore.model);
  });

  const login = async (email, password) => {
    await pb.collection('users').authWithPassword(email, password);
  };

  const register = async (email, password, name) => {
    await pb.collection('users').create({
      email,
      password,
      passwordConfirm: password,
      name
    });
  };

  const logout = () => {
    pb.authStore.clear();
  };

  return (
    <AuthContext.Provider value={{ user, login, register, logout }}>
      {children}
    </AuthContext.Provider>
  );
}

export function useAuth() {
  return useContext(AuthContext);
}
```

## 用户权限管理

### 角色字段

创建一个角色字段来管理用户权限：

```javascript
// 创建用户时设置角色
await pb.collection('users').create({
  email: 'admin@example.com',
  password: 'admin123',
  role: 'admin'
});

await pb.collection('users').create({
  email: 'user@example.com',
  password: 'user123',
  role: 'user'
});
```

### 基于角色的 API 规则

```
# 仅管理员可以创建
create = @request.auth.role == "admin"

# 管理员可以查看所有，用户只能查看自己的
view = (@request.auth.role == "admin" || @request.auth.id == id)

# 管理员可以更新所有，用户只能更新自己的
update = (@request.auth.role == "admin" || @request.auth.id == id)

# 仅管理员可以删除
delete = @request.auth.role == "admin"
```

```javascript
// 管理员登录
const adminData = await pb.admins.authWithPassword(

  'admin_password'
);

// 检查管理员状态
if (pb.authStore.model?.collectionName === 'admins') {

}
```

## 故障排除

### 常见问题

1. **登录失败: Invalid credentials**
   - 检查邮箱和密码是否正确
   - 确认用户已创建

2. **未验证邮箱错误**
   - 确保邮箱验证已通过
   - 或者禁用邮箱验证要求

3. **令牌过期**
   - 使用 `authRefresh()` 刷新令牌
   - 重新登录

4. **OAuth2 回调失败**
   - 检查 OAuth2 配置
   - 确认重定向 URL 正确

## 安全最佳实践

1. **密码安全**
   - 始终使用 HTTPS
   - 实施强密码策略
   - 考虑使用密码哈希和加盐

2. **令牌管理**
   - 不要在客户端代码中硬编码令牌
   - 定期刷新令牌
   - 在登出时清除令牌

3. **邮箱验证**
   - 生产环境启用邮箱验证
   - 使用可靠的邮件服务

4. **速率限制**
   - 对登录和注册端点实施速率限制
   - 防止暴力破解攻击

5. **会话管理**
   - 设置合理的令牌过期时间
   - 实施适当的登出流程
