# Wiki.js 资产上传文档

## 概述

Wiki.js 的资产管理系统提供了完整的文件上传、存储和管理功能。资产包括图片、文档、视频等各种文件类型。

## 认证和授权

资产上传需要适当的权限：
- **write:assets**: 上传和编辑资产的权限
- **manage:system**: 管理员系统权限（完全访问）

### 权限验证流程
1. 系统验证用户身份
2. 检查用户是否具有上传权限
3. 验证目标文件夹的访问权限
4. 执行上传操作

## 文件上传方式

### 1. Web 界面上传
通过管理界面的资产管理部分上传：
- **拖放上传**: 直接拖放文件到上传区域
- **点击选择**: 传统文件选择方式
- **批量上传**: 支持多文件同时上传

### 2. API 上传
使用 REST API 上传文件：

```http
POST /api/assets/upload
Content-Type: multipart/form-data
Authorization: Bearer <token>

file: <binary>
folder: /path/to/folder
```

### 3. 编辑器内上传
在页面编辑器中直接插入：
- **图片**: 从剪贴板粘贴或拖放
- **文件**: 通过资产浏览器选择
- **URL**: 外部资源链接

## 存储后端

Wiki.js 支持多种存储后端：

### 1. 本地文件系统
默认存储选项，文件保存在服务器本地：
```javascript
{
  "storage": {
    "type": "local",
    "config": {
      "path": "./data/assets"
    }
  }
}
```

### 2. 云存储服务

#### Amazon S3
```javascript
{
  "storage": {
    "type": "s3",
    "config": {
      "region": "us-east-1",
      "bucket": "my-wiki-assets",
      "accessKeyId": "YOUR_KEY",
      "secretAccessKey": "YOUR_SECRET"
    }
  }
}
```

#### Azure Storage
```javascript
{
  "storage": {
    "type": "azure",
    "config": {
      "account": "storageaccount",
      "container": "wiki-assets",
      "sasKey": "YOUR_SAS_KEY"
    }
  }
}
```

#### Google Cloud Storage
```javascript
{
  "storage": {
    "type": "gcs",
    "config": {
      "bucket": "wiki-assets",
      "keyFilename": "./path/to/keyfile.json"
    }
  }
}
```

## 文件夹组织

### 层级结构
资产文件夹支持多层嵌套结构：
```
assets/
├── images/
│   ├── logos/
│   └── screenshots/
├── documents/
│   ├── pdf/
│   └── office/
└── videos/
```

### 路径访问
- 资产通过路径访问：`/images/logos/company.png`
- 路径反映文件夹层级结构
- 支持相对路径引用

### 文件夹权限
可以在文件夹级别应用访问控制：
- **读取权限**: 谁可以查看文件夹内容
- **写入权限**: 谁可以上传到此文件夹
- **管理权限**: 谁可以管理文件夹结构

## 文件处理

### 自动处理功能
1. **图片优化**: 自动压缩和优化图片
2. **缩略图生成**: 为图片创建缩略图版本
3. **格式转换**: 支持常见格式转换
4. **病毒扫描**: 可选的上传文件安全扫描

### 文件限制
- **大小限制**: 配置最大文件大小
- **类型限制**: 限制允许的文件类型
- **命名规范**: 自动处理文件名冲突

## 资产元数据

每个资产包含以下元数据：
- **文件名**: 原始文件名
- **MIME 类型**: 文件类型标识
- **文件大小**: 字节大小
- **上传者**: 上传用户信息
- **上传时间**: 时间戳
- **文件夹位置**: 存储路径
- **URL**: 访问地址
- **缩略图**: 预览图片URL

## 安全考虑

### 访问控制
- 资产受与 wiki 相同的身份验证和授权系统保护
- 支持基于路径的访问控制
- 可以设置公开或私有资产

### 文件验证
- **类型检查**: 验证文件扩展名和 MIME 类型
- **内容检查**: 验证文件内容与类型匹配
- **大小验证**: 强制执行文件大小限制

### 存储安全
- **加密存储**: 支持静态数据加密
- **安全传输**: 强制 HTTPS 连接
- **备份策略**: 自动备份重要资产

## 常见问题

### 上传失败
1. 检查文件大小限制
2. 验证文件类型是否允许
3. 确认用户权限
4. 检查存储空间

### 图片显示问题
1. 验证存储后端配置
2. 检查 URL 生成设置
3. 确认 CDN 配置（如使用）

### 性能问题
1. 考虑使用云存储后端
2. 启用 CDN
3. 优化图片大小
4. 实施缓存策略

## 最佳实践

1. **组织结构**: 使用清晰的文件夹层次结构
2. **命名规范**: 使用描述性的文件名
3. **版本管理**: 保留重要文件的版本
4. **定期清理**: 删除未使用的资产
5. **备份策略**: 定期备份重要资产
6. **监控使用**: 跟踪存储使用情况
7. **安全配置**: 适当配置访问权限
