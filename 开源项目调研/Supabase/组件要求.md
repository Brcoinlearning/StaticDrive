# Supabase 组件要求和配置

## 系统最低要求

### 硬件要求

运行完整的Supabase组件（包括所有服务）：

| 资源 | 最低配置 | 推荐配置 |
| --- | --- | --- |
| 内存 (RAM) | 4 GB | 8 GB+ |
| CPU | 2核 | 4核+ |
| 磁盘空间 | 50 GB SSD | 80 GB+ SSD |

### 说明

- 上述要求适用于开发和中小型生产工作负载
- 大型生产环境可能需要更高配置
- SSD硬盘对于数据库性能至关重要
- 内存不足会严重影响性能

### 资源优化

如果不需要特定服务，可以从`docker-compose.yml`中移除以减少资源需求：

- Logflare (Analytics)
- Realtime
- Storage
- imgproxy
- Edge Runtime (Functions)

## 核心组件

### 1. Studio (管理仪表板)

**功能：**
- 自托管Supabase项目的可视化管理界面
- 数据库表管理
- SQL查询编辑器
- 用户和权限管理
- API密钥管理
- 访问日志和监控

**访问方式：**
- 端口：8000（通过API网关）
- URL：`http://<your-domain>:8000`
- 认证：HTTP Basic Authentication

**配置变量：**
- `DASHBOARD_USERNAME`: Studio用户名
- `DASHBOARD_PASSWORD`: Studio密码

### 2. Kong (API网关)

**功能：**
- API请求路由
- 认证和授权
- 速率限制
- 请求/响应转换
- 安全策略执行

**端口：**
- 8000 (HTTP)
- 8443 (HTTPS)

**重要性：**
- 所有API请求的统一入口点
- 提供安全层和流量管理

### 3. Auth (GoTrue)

**功能：**
- 基于JWT的身份验证API
- 用户注册和登录
- 会话管理
- 密码重置
- OAuth集成
- 多因素认证（MFA）

**配置要求：**
- `JWT_SECRET`: JWT签名密钥（必须保密）
- `ANON_KEY`: 客户端API密钥
- `SERVICE_ROLE_KEY`: 服务端API密钥（完全访问权限）
- `SITE_URL`: 认证重定向URL

**SMTP配置（生产环境必需）：**
- `SMTP_HOST`: SMTP服务器地址
- `SMTP_PORT`: SMTP端口
- `SMTP_USER`: SMTP用户名
- `SMTP_PASS`: SMTP密码
- `SMTP_ADMIN_EMAIL`: 管理员邮箱
- `SMTP_SENDER_NAME`: 发件人名称

### 4. PostgREST (REST API)

**功能：**
- 将Postgres数据库转换为RESTful API
- 自动生成API端点
- 支持CRUD操作
- 与RLS集成
- 支持复杂查询和关联

**配置要求：**
- `JWT_SECRET`: 与Auth服务共享
- `PGRST_DB_SCHEMA`: API暴露的数据库模式
- `PGRST_DB_ANON_ROLE`: 匿名角色

### 5. Realtime

**功能：**
- Elixir服务器
- 监听Postgres数据库变更
- 向订阅客户端广播变更
- 实时数据同步
- WebSocket支持

**配置要求：**
- `SECRET_KEY_BASE`: Realtime通信加密密钥
- 端口：4000（内部）

### 6. Storage

**功能：**
- 文件存储和管理的RESTful API
- S3兼容性
- 支持多种上传协议
- 图像转换
- CDN集成

**配置要求：**
- `S3_PROTOCOL_ACCESS_KEY_ID`: S3协议访问密钥ID
- `S3_PROTOCOL_ACCESS_KEY_SECRET`: S3协议密钥
- `MINIO_ROOT_PASSWORD`: MinIO管理员密码
- 存储后端配置（本地文件或S3兼容服务）

**存储类型：**
- 文件存储桶
- 分析存储桶
- 向量存储桶

### 7. imgproxy

**功能：**
- 快速安全的图像处理服务器
- 实时图像转换
- 优化和压缩
- 多种图像格式支持

**特点：**
- 内存高效的图像处理
- 支持远程图像URL
- 安全的图像处理

### 8. postgres-meta

**功能：**
- Postgres管理的RESTful API
- 获取表结构
- 添加角色
- 运行查询
- 数据库元数据管理

**配置要求：**
- `PG_META_CRYPTO_KEY`: 连接字符串加密密钥

### 9. Postgres (数据库)

**功能：**
- 核心关系型数据库
- 数据持久化
- 事务处理
- 扩展支持
- RLS策略执行

**配置要求：**
- `POSTGRES_PASSWORD`: 数据库密码
- `POSTGRES_DB`: 数据库名称
- `POSTGRES_USER`: 数据库用户
- 端口：5432（内部）

**默认角色：**
- postgres: 超级用户
- supabase_admin: 管理员角色
- supabase_auth: Auth服务角色
- supabase_storage: Storage服务角色
- authenticated: 已认证用户角色
- anon: 匿名用户角色

### 10. Supavisor (连接池)

**功能：**
- Postgres连接池化
- 高效连接管理
- 支持事务和会话模式
- 负载均衡

**端口：**
- 5432: 会话模式（类似直接Postgres连接）
- 6543: 事务模式（使用连接池）

**配置要求：**
- `POOLER_TENANT_ID`: 租户ID

### 11. Edge Runtime (Deno)

**功能：**
- 基于Deno的Web服务器
- Edge Functions执行环境
- JavaScript/TypeScript支持
- WASM支持

**配置要求：**
- 函数存储路径：`volumes/functions`
- 端口配置

### 12. Logflare (Analytics)

**功能：**
- 日志管理和事件分析
- 日志摄取和查询
- 实时日志流
- 事件分析

**配置要求：**
- `LOGFLARE_PUBLIC_ACCESS_TOKEN`: 日志摄取和查询API令牌
- `LOGFLARE_PRIVATE_ACCESS_TOKEN`: Logflare管理API令牌

### 13. Vector

**功能：**
- 高性能可观察性数据管道
- 日志收集和转发
- 与Logflare集成

## 可选组件

### Analytics (分析工具)

## 安全要求

### 密钥管理

**必需的安全密钥：**

1. **JWT_SECRET**
   - 用于JWT签名和验证
   - 必须保密且足够长
   - Auth、PostgREST等服务共享

2. **数据库密钥**
   - `POSTGRES_PASSWORD`: 数据库访问
   - 必须强壮且唯一

3. **加密密钥**
   - `SECRET_KEY_BASE`: Realtime通信加密（至少64字符）
   - `VAULT_ENC_KEY`: Supavisor配置加密（正好32字符）
   - `PG_META_CRYPTO_KEY`: Studio连接加密（至少32字符）

### API密钥

1. **ANON_KEY**
   - 客户端使用
   - 有限权限
   - 可以公开

2. **SERVICE_ROLE_KEY**
   - 服务端使用
   - 完全访问权限
   - 绝对不能暴露

### 生产环境安全

- 使用HTTPS（配置反向代理）
- 强密码策略
- 定期更新密钥
- 实施访问控制
- 监控安全日志
- 备份策略

## 推荐的密钥生成方法

```bash
# JWT_SECRET
openssl rand -base64 64

# SECRET_KEY_BASE (至少64字符)
openssl rand -base64 48

# VAULT_ENC_KEY (正好32字符)
openssl rand -hex 16

# PG_META_CRYPTO_KEY (至少32字符)
openssl rand -base64 24

# LOGFLARE令牌 (至少32字符)
openssl rand -base64 24

# S3协议密钥
openssl rand -hex 16  # ACCESS_KEY_ID
openssl rand -hex 32  # ACCESS_KEY_SECRET

# MINIO密码
openssl rand -hex 16

# 数据库密码
# 使用字母和数字，避免特殊字符
```

## 总结

Supabase自托管需要考虑以下关键方面：

1. **硬件资源**：确保足够的CPU、内存和存储
2. **服务配置**：正确配置所有必需的环境变量
3. **安全措施**：实施适当的安全策略和密钥管理
4. **网络设置**：配置DNS、防火墙和端口
5. **维护计划**：建立监控、备份和更新流程
6. **扩展性**：考虑未来增长和扩展需求

通过遵循这些要求和最佳实践，可以成功部署和维护一个生产就绪的Supabase自托管实例。
